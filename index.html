Learn

I'm surprised I even made a "Learn" section on this website, well now that it's here, I might as well throw some wisdom and knowledge, so here's what I'll suggest since I used them myself to hone my skillset in the past couple of years. For the ones that don't know, I'm a cybersecurity enthusiast, and this entire section is going to document the things I've learned over time in an organized structure, since I didn't follow a structured path and was confused myself, I'd like to guide others to a treasure I didn't possess.

Fundamentals:

Learning a specific skill has many approaches to it, there is no best way to do it, try not to get caught up trying to be perfect while learning. Feel free to make mistakes, but when you do, you must learn to get back up quickly and learn what caused it, so you don't do it twice.

Firstly, to learn any subject or topic, it's always good to do it in a proper methodology, be it your own or any proven method you find online.

Lookup The Pomodoro Technique or The Feynman Technique. There are a lot more techniques and methods to pick, choose what works for you, and customize if needed.

Secondly, be consistent. It takes time to be good at any skill you are passionate about; contrary to the popular 10,000-hour rule, you can NOT become a grandmaster in a discipline if you spend 10,000 hours just doing it with no quantifiable goals. Instead, you will have to spend 10,000 hours working like you desire to be a grandmaster in the discipline. There is a big difference.

Programming:

Staring off with programming, of course, everyone I see that wants to get into programming starts only with either DSA or a programing language by watching some tutorials online and feeling great about it, and to me, that's perplexing. The best way to learn to program is to learn DSA and a programming language simultaneously while also building projects with it, so you know how to put things together. Start learning a language like C if you're patient, if not, then learn something with a simpler syntax like JavaScript while learning DSA on the side. Assembly language will be a great addition to your toolbox since you will be able to directly manipulate data at the computer architectural level, which can be useful in some specific use cases like exploit development or even while doing reverse engineering sometimes. (I've provided the links if anyone wants it.)

A bonus skill is to learn the right words so you know what to Google when you're facing an issue. Googling is a whole new skill by itself, and while being a self-learner, it's one of the most important things you'll have to know. I learned this the hard way. Check out this video, I like how comprehensive it is.

Linux/UNIX:

Once you're comfortable with code, get your feet wet with any UNIX-Like Operating System because most of the web servers online use Linux as the Operating System, understand how it works, and familiarize yourself with some basic underlying functions like memory allocation, and others (These will be helpful later on while developing exploits). If I'm honest it's just that it's a whole lot better than any other Operating systems I've come across, especially Arch Linux, here's a treat. If you're all soaked up with frustration right at the start, congratulations, you're doing it right.

Networking:

The next step is to start learning networking, take it really slow. Understanding the basics will lay the foundation for your future endeavors (if any). Start with the basic IP/TCP model and move on from there to learn more and more about other Internet Protocols, understanding various concepts like routers, cryptography, and many more will make you feel more comfortable while moving forward. Learn what the TOR network is, what I2P is, SOCKS5, and a lot of other anonymizing protocols which you can use to just stay safe while you're "browsing" the web.

While learning networking, an issue I faced was that I forgot a lot of things in just a week or so after learning, and the solution to that is to learn through tiny bits of information, and review them later on repeatedly, for which you will have to maintain good notes. I used Notepad++ at the start to maintain the notes I prepared, but later on, I came across Notion, and trust me it's a student's wet dream.

A tip to learn networking at a higher and deeper level is to learn socket programming in C because you will explicitly be able to see what's going on under the hood, and of course, practice and patience will be the essence to success.
